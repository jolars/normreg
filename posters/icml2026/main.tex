  % first column -- the problem

  \begin{minipage}[t][\colheight][t]{\onecolwidth}
    \begin{alertblock}{Take-Home Message}
      Normalization is \textbf{not} neutral preprocessing. For
      \alert{binary features}, the \emph{class balance} (proportion of ones)
      leaks into your lasso, ridge, and elastic net coefficients---and
      \emph{which} normalization you pick decides whether it does.
      \alert{The good news:} the right choice is predictable---match the scaling
      (\(\delta\)) to your penalty and goal, or tune it.
    \end{alertblock}

    \begin{block}{The Problem}
      Regularized models are scale-sensitive, so we \emph{normalize} (center and
      scale) features first. But there are many ways to do this, and for binary
      features there is no obvious choice. Despite this, the effect of that
      choice has gone unstudied.

      \medskip

      Below: lasso coefficient paths for a real dataset (\data{w1a}) under two
      normalizations. Standardization makes several features stand out, while
      max--abs flattens them---the normalization choice alone decides which
      features look important.
      \begin{figure}
        \centering
        \includegraphics{realdata_paths.pdf}
        \caption{
          Lasso paths for \data{w1a} under standardization (\(\delta=\tfrac12\))
          and max--abs (no scaling, \(\delta=0\)), with the first features to
          enter highlighted. The normalization alone changes which features
          dominate.
        }
      \end{figure}
    \end{block}

    \begin{block}{Setup}
      We study the elastic net~(Zou \& Hastie, 2005),
      \[
        \tfrac{1}{2}\lVert\vec y - \beta_0 - \tilde{\mat X}\vec\beta\rVert_2^2 + \lambda_1\lVert\vec\beta\rVert_1 + \tfrac{\lambda_2}{2}\lVert\vec\beta\rVert_2^2,
      \]
      which gives the \alert{lasso} (\(\lambda_2=0\)) and \alert{ridge}
      (\(\lambda_1=0\)) as special cases. Each binary feature \(\vec x_j\) is
      centered and scaled by \(s_j = (q_j - q_j^2)^\delta\), \(\delta\geq 0\),
      covering no scaling (\(\delta=0\)), standard deviation
      (\(\delta=\tfrac12\)), and variance (\(\delta=1\)).
    \end{block}
  \end{minipage}
  \hfill
  %
  % second column -- the theory
  %
  \begin{minipage}[t][\colheight][t]{\onecolwidth}
    \begin{block}{Class Balance Enters the Estimate}
      With orthogonal features and no noise, the estimate on the original scale
      is
      \[
        \hat\beta_j = \frac{\st_{\lambda_1}\!\left(\dfrac{\beta_j^*\, n\,\amath{(q_j-q_j^2)}}{s_j}\right)}
             {s_j\left(\dfrac{n\,\amath{(q_j-q_j^2)}}{s_j^2} + \lambda_2\right)} .
      \]
      The feature variance \(\amath{q_j-q_j^2}\) couples class balance to the
      estimate, mediated entirely by the scale \(s_j\). The right \(\delta\)
      makes it cancel:

      \bigskip

      \begin{table}
        \centering
        \renewcommand{\arraystretch}{1.3}
        \begin{tabular}{@{}lll@{}}
          \toprule
          \textbf{Penalty} & \textbf{Cancelling Scale} & \(\symbf{\delta}\) \\
          \midrule
          Lasso            & variance                  & \(1\) \\
          Ridge            & standard dev.             & \(\tfrac12\) \\
          Elastic net      & \alert{none exists}       &  \\
          \bottomrule
        \end{tabular}
      \end{table}

      \bigskip

      For the elastic net, \emph{no} \(s_j\) removes the bias---instead weight
      the \emph{penalty} (\(u_j=v_j=(q_j-q_j^2)^\omega\), \(\omega=1\)).
    \end{block}

    \begin{block}{Selection Follows \(\delta\)}
      Lower \(\delta\) means imbalanced features are far less likely to be
      selected; as \(q\to1\), selection probability \(\to 0\) for
      \(\delta<\tfrac12\) and \(\to 1\) for \(\delta>\tfrac12\).
      \begin{figure}
        \centering
        \includegraphics{selection_probability.pdf}
        \caption{
          Lasso selection probability vs.\ class balance \(q\) for several
          \(\delta\). Dotted line: asymptotic limit at \(\delta=\tfrac12\).
        }
      \end{figure}
    \end{block}
  \end{minipage}
  \hfill
  %
  % third column -- trade-off and extensions
  %
  \begin{minipage}[t][\colheight][t]{\onecolwidth}
    \begin{block}{Beyond Orthogonality}
      The class-balance bias is not just an artifact of the orthogonal
      theory---it persists with many noisy, correlated features. With 20 signals
      whose class balance rises \(0.5\to0.99\), no scaling (\(\delta=0\))
      shrinks the most imbalanced signals to zero, while variance scaling
      (\(\delta=1\)) recovers them.
      \begin{figure}
        \centering
        \includegraphics{binary_decreasing.pdf}
        \caption{
          Lasso estimates (mean \(\pm\) SD) for a simulation with \(p=1000\)
          (the first 20 features are signals with class balance \(0.5\to0.99\);
          first 30 shown). No scaling drops the imbalanced signals; variance
          scaling recovers them.
        }
      \end{figure}
    \end{block}

    \begin{block}{Tuning \(\delta\)}
      Larger \(\delta\) removes class-balance bias but inflates variance, so
      there is no universal best \(\delta\). On real data, the cross-validated
      error is minimized at an \emph{interior} \(\delta\) that differs by
      dataset---so treat \(\delta\) as a hyperparameter and tune it.
      \begin{figure}
        \centering
        \includegraphics{hyperopt_surfaces.pdf}
        \caption{
          Hold-out (validation) NMSE over \((\lambda, \delta)\) for the lasso on
          three datasets. White dot: tuned optimum; dotted line: best \(\delta\)
          at each \(\lambda\). The optimal \(\delta\) is interior and
          dataset-dependent.
        }
      \end{figure}
    \end{block}

  \end{minipage}

    \medskip

    \begin{alertblock}{Recommendations}
      \begin{description}
        \item[Selection] If missing a feature is costly, scale by
              \emph{variance} (\(\delta=1\)).
        \item[Prediction] Standard deviation or no scaling; better, \emph{tune}
              \(\delta\) as a hyperparameter.
        \item[Elastic net] Weight the \emph{penalty} (\(\omega=1\)), not the
              features.
      \end{description}
    \end{alertblock}

    \begin{block}{Why It Matters}
      The two most common defaults are \emph{both} biased: standardization (the
      \pkg{glmnet} default; Friedman et al., 2010) and no scaling (recommended
      for sparse data in \pkg{scikit-learn}). The class-balance effect is
      therefore likely pervasive in published analyses.
    \end{block}
